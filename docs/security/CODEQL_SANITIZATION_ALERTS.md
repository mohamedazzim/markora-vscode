## Alert #2 data flow and root cause

The original flow was:

```text
Markdown heading text
→ headingAnchors(source)
→ append(text)
→ text.replace(/<[^>]*>/g, '')
→ slug/ID string
→ outline labels and heading-fragment lookup
```

The regular expression removed a multi-character tag-shaped sequence. Crafted
overlapping input could leave a new tag-shaped sequence after the replacement.
The resulting value was not assigned to `innerHTML`, but it was still an
incomplete sanitizer and could produce misleading anchor identifiers or become
unsafe if a downstream consumer later rendered the ID as markup. This is a
genuine correctness and defense-in-depth issue, not a dismissed false positive.

The fix delegates tag removal to `sanitize-html` with an empty tag and attribute
allowlist, then applies the existing slug normalization. Sanitization now has a
parser-aware boundary instead of a multi-character blacklist replacement.

## Alert #1 data flow and root cause

The original flow was:

```text
HTML/Tiptap <img alt="...">
→ structuredHtmlToMarkdown
→ alt.replace(/([\[\]])/g, '\\$1')
→ ![escaped alt](destination)
→ Markdown serializer / TextDocument update
```

The replacement escaped brackets but left backslashes untouched. A backslash in
untrusted alt text could therefore change how a later bracket or delimiter was
interpreted in Markdown. The fix uses one global context-specific replacement
for backslashes and brackets (`/[\\[\]]/g`), so every metacharacter is escaped in
one pass and no subsequent replacement can expose a new escape boundary.

## Related hardening

`sanitizeMarkdownHtml` now applies tag-specific URL schemes and rejects data
URLs that are not base64-encoded images. `http`, `https`, and `mailto` remain
available for links; `data:text/html` and similar executable image payloads are
removed. Existing relative-resource placeholders, strict Mermaid rendering,
webview CSP, Zod message validation, Workspace Trust, and external-link policy
remain unchanged.

## Regression coverage

`packages/markdown-core/tests/security-sanitization.test.ts` covers:

- overlapping/repeated angle-bracket and script-like sequences;
- backslashes, brackets, Unicode, and safe image paths;
- JavaScript, HTML, SVG event-handler, and `data:text/html` payloads;
- encoded/double-encoded values;
- Mermaid arrows and HTML-like labels;
- KaTeX backslashes and entities;
- repeated transformation stability.

The focused suite contains 5 tests. The broader unit run contains 75 tests.

## Review and release decision

The findings affect the shared Markdown core used by the distributed VS Code
webview and HTML export, so a patch release is required after the fix is merged.
No alert was dismissed. Marketplace publication requires the existing
publisher-owned `vsce` credential and will not be claimed until authentication
and a clean-profile release verification succeed.
